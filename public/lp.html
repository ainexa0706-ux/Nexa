ワークスペース

会話UIと開発UIを切り替えられる。

チャットモードは会話と文章作成に集中し、コードモードはPC上の作業フォルダーをプロジェクトとして扱います。 両方モードなら、相談からファイル作成までをひとつの流れで進められます。モードはチャットごとに保持されます。

チャットモード

中央入力からすぐ開始

新規チャットは画面中央から始まり、会話開始後は入力欄が下部へ移動します。

コードモード

選択フォルダーへ直接保存

必要な数のファイルを作成・編集・削除し、変更ログ、レビュー、検証結果を会話内に残します。

記憶

前回の続きを復元

プロジェクト単位の履歴とメモリーで、チャットが途切れても作業の文脈を維持します。

ジャンルを限定しない開発

一つのテンプレートではなく、依頼から構成を決める。

Nexa 3.0は最新の依頼と現在のフォルダーを毎回読み直します。同じジャンルでも別の依頼なら前回の成果物を使い回さず、 プロジェクト名、機能、ファイル構成を組み直します。

Web

サイトとWebアプリ

LP、管理画面、業務ツール、ダッシュボード、インタラクティブUIをレスポンシブ対応で構築します。

Game

2D・3Dゲーム

操作、ループ、敵、スコア、HUDを含むゲームを作成。3Dシューティングは専用の実行可能な復旧構成も備えます。

Backend

APIとサーバー

REST API、データ処理、Node.jsバックエンド、設定ファイル、テスト、READMEまで必要に応じて分割します。

Tools

CLIと自動化

Python、JavaScript、C#、Java、Go、Rustなど、依頼に合う言語と構成でツールを作成します。

Existing code

既存プロジェクトを継続編集

初回だけで終わりません。同じチャットの2回目以降も既存コードを再読込し、修正・追加・削除を続けます。

Recovery

保存形式を自動補修

モデル出力が壊れた場合は形式を再生成し、それでも失敗した時は依頼別の独立フォルダーへ安全に復旧します。

コードモード

依頼を受けるたび、フォルダーの今を確認する。

最新のメッセージを新しい実装ターンとして扱い、古い出力より優先します。既存フォルダーに別プロジェクトを作る場合は、 固有名のサブフォルダーへ分離して元のファイルを保護します。

開発フロー

Nexa 3.0が役割を分けて品質を確認する。

計画、記憶、ツール選択、実装、検証、安全確認、最終回答を分離します。短い指示は直近の会話へつなげ、 確認が必要な場合だけ選択式の質問を表示します。回答後の質問カードは残りません。
